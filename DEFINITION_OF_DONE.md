# Definition of Done — ledger

Instantiates `QUALITY-AND-METRICS-STANDARD.md`'s per-repo Definition of Done
(QM-18) for ledger's actual shape: a Python CLI + self-hosted web archive
server, bilingual (EN/ES) UI, no AI/LLM component, no maintainer-operated
production deployment (see `docs/DORA-DELIVERY-HEALTH-REVIEW.md`, QM-11, for
why that last point matters to how "done" is measured over time). Reviewed
quarterly alongside the DORA review; update this file in the same PR that
changes what "done" means (e.g. adding a new gate).

CODEOWNER-protected: `.github/CODEOWNERS` routes every path, and `/.github/`,
`/src/ledger/access/`, and `/src/ledger/identity.py` explicitly, to
`@ChelseaKR`. Modifying this file therefore already requires the same review
the standard asks for; a distinct branch-protection ruleset making that
review *mandatory* (rather than requested) is the separate, server-side gap
tracked below under "Branch protection."

## AUTO-GATE (CI on every PR; `make verify` reproduces every item below locally)

1. **Format + lint** — `ruff check` + `ruff format --check` on `src/` and
   `tests/`, zero errors (`ci.yml` lint job; `make lint`).
2. **Type-check** — `mypy` strict config, zero errors (`ci.yml`; `make type`).
3. **Tests + coverage** — full suite green on Python 3.11–3.13; branch
   coverage ≥ 85% (`--cov-fail-under=85`, `[tool.coverage.run] branch = true`
   in `pyproject.toml`); cyclomatic complexity ≤ 10 (`ruff` `C901`,
   `max-complexity = 10`) — 7 pre-existing functions are waived with dated
   `# noqa: C901` comments pending a deliberate split, tracked in
   `docs/ROADMAP.md` (CQ-05).
4. **Security** — CodeQL (`python`) and gitleaks (secret scan) run on every
   PR; `pip-audit` is blocking with no mute pattern
   (`SECURITY-AND-SUPPLY-CHAIN-STANDARD` §4 forbids `|| true`-ing this gate);
   every workflow `uses:` is SHA-pinned; the self-host container image is
   scanned for CRITICAL/HIGH CVEs with Trivy. **Not yet gated:** Semgrep SAST,
   a scheduled full-history TruffleHog scan, and CycloneDX SBOM + cosign +
   SLSA provenance on a release artifact — tracked in `docs/ROADMAP.md`
   (SEC-07, SEC-19, REL-08/P1-6; open PRs already in flight for the first
   two).
5. **Workflow SAST** — **not yet gated.** No zizmor job exists; CodeQL does
   not analyze `language: actions`. Tracked in `docs/ROADMAP.md` (CICD-19/20,
   P1-3).
6. **Accessibility** — WCAG 2.2 AA, enforced today via
   `ledger.accessibility_check` (structural checks: lang/landmarks/contrast/
   alt-text) over the built `web/` surface, run as a required CI job
   (`ci.yml` "accessibility gate"), plus a committed
   `docs/accessibility/ACR.md` (VPAT 2.5, regenerated by `make acr`).
   **Not yet gated:** axe-core / Lighthouse / pa11y / Playwright
   keyboard+reflow automation — the structural checker substitutes for these
   today. Tracked in `docs/ROADMAP.md` (A11Y-01–03/07/09, P3-7).
7. **i18n (EN/ES, bilingual)** — `make i18n`, required in CI: POT extraction
   is current (fails the build on drift), EN/ES key-parity and
   placeholder-parity, both catalogs compile clean (`msgfmt --check
   --check-format --check-domain`), every authored locale tag is valid
   BCP-47.
8. **AI-eval** — N/A. Ledger has no AI/LLM/RAG component
   (`docs/adr/0006-standards-applicability.md`).
9. **Observability** — Tier C (structured-log opt-in, not a full OTel/SLO
   stack); no CI gate today beyond that declaration
   (`README.md#observability`). No `--log-format json` gate exists yet.
10. **Performance** — no k6/Lighthouse budget gate exists yet. Tracked in
    `docs/ROADMAP.md` (QM-02, P3-5).
11. **Build** — the self-host container image builds and is scanned in
    `ci.yml`'s container job on every PR. A tag-triggered release workflow
    (sdist/wheel + signed, provenanced artifacts) does not exist yet —
    tracked in `docs/ROADMAP.md` (REL-08, P1-6); CHANGELOG's `0.1.0` claim is
    corrected pending that real cut (REL-03, P2-6).

`make verify` runs stages 1, 2, 3, 6, 7, and the `pip-audit`/`gitleaks` half
of stage 4 locally, byte-for-byte identical to what CI requires

`docs/ROADMAP.md` row naming the gap and the item that closes it.

## REVIEW-GATE (human sign-off, committed as PR attestation + artifact)

- PR description states acceptance criteria; `.github/PULL_REQUEST_TEMPLATE.md`
  (where present) or the PR body names the ISO 25010 characteristic(s) the
  change targets.
- Any change touching `.github/workflows/`, `src/ledger/access/`, or
  `src/ledger/identity.py` routes through `@ChelseaKR` via CODEOWNERS — the
  disclosure/no-outing choke point and the identity vault are the two places
  a bug is a safety incident, not just a defect (`docs/THREAT-MODEL.md`).
- A new custom interactive surface in `web/` gets a keyboard + screen-reader
  pass before merge — informal today, since no dated walkthrough artifact
  exists yet (tracked in `docs/ROADMAP.md`, A11Y-11/12/16/18, P2-3).
- `CHANGELOG.md` is updated when user-visible behavior changes
  (`CONTRIBUTING.md`).
- A new external attack surface gets a `docs/THREAT-MODEL.md` update before
  merge, not after.

## RELEASE-GATE

Ledger has not yet cut a tagged release (`git tag` is empty; REL-03/REL-08,
`docs/ROADMAP.md`). Once the release workflow (P1-6) lands, this section
governs every tag:

- `make verify` is green at the tagged commit.
- Tag and `pyproject.toml` version agree; `CHANGELOG.md` has a section for
  the released version.
- SBOM + cosign signature + SLSA provenance are generated as part of the
  publish workflow, never hand-built.
- The published artifact is re-downloaded and its attestation/signature
  checked before the release is considered done.

## Branch protection

**Not yet enabled as a live GitHub ruleset** — this needs an interactive
GitHub Settings / `gh api repos/ChelseaKR/ledger/rulesets` change, which is
outside what this repo's files can express or what an automated remediation
pass should do unattended (`docs/ROADMAP.md`, CICD-12/P2-4). The ruleset this
Definition of Done assumes, once enabled: PR required (≥1 approval, ≥2 for
changes to `src/ledger/access/` or `identity.py`), last-pusher cannot
self-approve, stale reviews dismissed on new pushes, CODEOWNERS review
required on the routed paths above, required status checks in strict mode
covering every AUTO-GATE job that exists today, signed commits, linear
history, no force-push, no admin bypass on `main`.
